üë§ Users

1. Sign Up

Create a new user account.

POST /users/signup

Request Body:

{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "password123"
}

Response (201):

{
  "status": "success",
  "message": "User registered successfully",
  "data": {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com",
    "role": "user"
  }
}

Example with Live API:

curl -X POST https://ecommerce-backend-peach-six.vercel.app/users/signup \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "password123"
  }'

2. Login

Authenticate user and receive JWT token.

POST /users/login

Request Body:

{
  "email": "john@example.com",
  "password": "password123"
}

Response (200):

{
  "status": "success",
  "message": "Login successful",
  "data": {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com",
    "role": "user",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}

Example with Live API:

curl -X POST https://ecommerce-backend-peach-six.vercel.app/users/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "password123"
  }'

3. Get Profile

Retrieve authenticated user's profile information.

GET /users/profile

REQUIRED Authorization: Bearer <token>

Response (200):

{
  "status": "success",
  "data": {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com",
    "role": "user",
    "createdAt": "2026-02-13T10:30:00.000Z"
  }
}

üõí Cart

1. Get Cart

Retrieve authenticated user's shopping cart.

GET /cart

REQUIRED Authentication

Try it:

curl -X GET https://ecommerce-backend-peach-six.vercel.app/cart \
  -H "Authorization: Bearer <your_token>"

2. Add to Cart

Add a product to the user's shopping cart.

POST /cart/add

REQUIRED Authentication

Request Body:

{
  "productId": 1,
  "quantity": 2
}

Try it:

curl -X POST https://ecommerce-backend-peach-six.vercel.app/cart/add \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your_token>" \
  -d '{
    "productId": 1,
    "quantity": 2
  }'

3. Remove from Cart

Remove a specific product from cart.

DELETE /cart/remove/:productId

REQUIRED Authentication

4. Clear Cart

Remove all items from cart.

DELETE /cart

REQUIRED Authentication

‚ö†Ô∏è Error Handling

The API uses meaningful HTTP status codes:

Status Code Meaning
200 OK - Request successful
201 Created - Resource created successfully
400 Bad Request - Invalid input/validation failed
401 Unauthorized - Authentication required or failed
403 Forbidden - Insufficient permissions
404 Not Found - Resource not found
500 Internal Server Error

üîê Authentication & Authorization

JWT Authentication

Token Format: Authorization: Bearer <token>
  • Tokens are generated on successful login
  • Token expiration: 1 hour
  • Verified by auth middleware on protected routes

Protected Routes

Authentication Required:
  • GET /users/profile
  • GET /cart
  • POST /cart/add
  • DELETE /cart/remove/:productId
  • DELETE /cart
  • POST /orders
  • GET /orders
Admin Authentication Required:
  • POST /products
  • PUT /products/:id
  • DELETE /products/:id
