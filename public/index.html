üîê Authentication

Token Info: Uses JWT. Pass token in header: Authorization: Bearer . Expiry: 1 hour.
POST /users/signup

Register a new user account.

Request Body
{
  "name": "Alex Smith",
  "email": "alex@example.com",
  "password": "securePassword123"
}
Response (201 Created)
{
  "status": "success",
  "message": "User registered successfully",
  "data": {
    "id": 1,
    "name": "Alex Smith",
    "email": "alex@example.com",
    "role": "user"
  }
}
POST /users/login

Authenticate a user and receive a JWT token.

Request Body
{
  "email": "alex@example.com",
  "password": "securePassword123"
}
Response (200 OK)
{
  "status": "success",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

üì¶ Products

GET /products

Retrieve a list of all products.

Response
{
  "status": "success",
  "count": 5,
  "data": [
    {
      "id": 1,
      "name": "Laptop",
      "description": "High performance...",
      "price": 999.99,
      "stock": 10
    }
  ]
}
GET /products/:id

Get details of a single product.

POST /products

ADMIN Create a new product.

Request Body
{
  "name": "Gaming Mouse",
  "description": "High precision...",
  "price": 49.99,
  "stock": 100
}
PUT /products/:id

ADMIN Update a product.

DELETE /products/:id

ADMIN Delete a product.

üõí Cart

GET /cart

AUTH Get current user's cart.

POST /cart/add

AUTH Add item to cart.

Request Body
{
  "productId": 1,
  "quantity": 2
}
DELETE /cart/remove/:productId

AUTH Remove specific item from cart.

DELETE /cart

AUTH Clear entire cart.

‚ö†Ô∏è Error Handling

The API returns consistent error responses.

Standard Error Format
{
  "status": "error",
  "message": "Descriptive error message",
  "code": 400
}
  • 400: Bad Request (Validation)
  • 401: Unauthorized
  • 403: Forbidden
  • 404: Not Found
  • 500: Server Error
